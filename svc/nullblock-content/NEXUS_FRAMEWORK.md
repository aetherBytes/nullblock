# N.E.X.U.S. Agent Framework

**The Nullblock way to measure autonomous intelligence**

---

## The Five Dimensions

### N — Network
**What it is**: Agent-to-agent coordination capability  
**Measures**: Can it orchestrate other agents? Can it operate in a mesh?  
**High-N Agents**: Coordinators, orchestrators, mesh navigators

*Content angle*: "Network effects start with agents that talk to each other."

### E — Execution
**What it is**: Task completion speed & reliability  
**Measures**: How fast does it work? How reliable is it?  
**High-E Agents**: Speed demons, critical path handlers, real-time traders

*Content angle*: "Fast execution beats perfect planning."

### X — Expansion
**What it is**: Learning, adaptation, capability growth  
**Measures**: Does it improve over time? Can it handle new scenarios?  
**High-X Agents**: Learners, adapters, evolving systems

*Content angle*: "The best agents don't stay the same. They get smarter."

### U — Utility
**What it is**: Practical value delivered per unit of cost  
**Measures**: ROI, impact, real-world usefulness  
**High-U Agents**: Practical builders, impact creators, efficiency experts

*Content angle*: "Utility beats hype. What's it actually worth?"

### S — Security
**What it is**: Verifiability, auditability, trustworthiness  
**Measures**: Can we verify its actions? Can we trust its decisions?  
**High-S Agents**: Validators, auditable systems, deterministic processors

*Content angle*: "Trust is earned through transparency."

NEXUS is ours. It's how Nullblock talks about agent quality.

---

## Content Guidelines for NEXUS

**DO:**
- Use NEXUS when explaining agent differences
- Reference specific dimensions ("High-E" = speed)
- Show how dimensions balance (N+S = trustworthy coordination)
- Make it game-like (stats, profiles, specializations)

**DON'T:**
- Confuse with SPECIAL (Fallout)
- Oversimplify (agents have all five, just different balances)
- Use it for non-agent topics (NEXUS is agent-specific)
- Make it too technical (keep it accessible)
